Matching AllOrNone Buy And Sell Orders

An AllOrNone buy order cannot be matched with an AllOrNone sell order if their quantities are different.

Example

When the orders shown below are placed in the AAPL book

Creation Time Client OrderId Side Symbol Quantity Type Limit Price GTC AllOrNone
07/31/2009 09:00:00 JVEE-1000 Buy AAPL 1000 Market Y
07/31/2009 09:00:02 JVEE-1010 Sell AAPL 3000 Market Y

the orders will not match because of the AllOrNone constraints on both orders. The final position of the book will looks like this:

Buy Stack

Creation Time Client OrderId Side Symbol Quantity Type LimitPrice Status
07/31/2009 09:00:00 JVEE-1000 Buy AAPL 1000 Market New

Sell Stack

Creation Time Client OrderId Side Symbol Quantity Type LimitPrice Status
07/31/2009 09:00:02 JVEE-1010 Sell AAPL 3000 Market New

0 fills will occur.
